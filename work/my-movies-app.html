Case Study: myMovies App

INTRODUCTION

myMovies is a web app fpr movie fans, that offers information about movies, directors, and genres. Users can create a profile, update their data, and collect their favorite movies. The project involved creating a RESTful API, building the databse with Mongo DB and the client-side using React/Redux and Bootstrap.

PROBLEM TO SOLVE + CONTEXT

The app was supposed to satisfy an interest in movies: discover movies by title and let the user collect movies and offer further information, like showing what other movies a director has made.
On the technical side the requirement for the app was to be dynamic and single page, be responsive.
myMovies was part of the web development course at CareerFoundry to build skills in React and Redux, connecting to a self created API and implement Bootstrap for the UI.

GOAL

Create a single page, responsive web app that handles dynamic contect and has a pleasing and intuitive UI.

APPROACH

1. Step: Create the Server Side

RESTful API using Node.js and Express
create database collections manually to learn the structure of the objects
set up database Mongo DB (kind of databse?)
HTTP methods for API access + CRUD methods for database access
testing the API with Postman

2. Step: Create the Client Side

created components with React and Redux
app routing in main view
registration and login views
displaying list of movies with a search function tha searches titles
movie view, director and genre views with detailed information
movie view has a button to add the movie to list of favorites
profile view with list of favorites and form to update user data

CHALLENGE + DURATION STEPS

CHALLENGE

the genreal structure of React and the use of props and state
how things are connected, how data is being passed, keeping the orientation
Redux and the use of the state in this case > sessions with mentor and real live React-developer

DURATION + NEXT STEPS
